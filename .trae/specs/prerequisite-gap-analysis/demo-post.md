## 1. Demo 简介

**SparkForge** 是一个面向独立开发者的 **创意信号雷达与 AI 自动化落地平台**（Web 应用），帮助用户从海量互联网信息中发现下一个爆款创意，并通过 AI 评估商业可行性，快速生成可运行的产品原型。

### 核心用户
- 独立开发者 / 自由职业者
- 中小型创业团队的产品负责人
- 寻找下一个项目方向的 Maker

### 核心功能

| 功能模块 | 说明 |
|---------|------|
| **创意雷达** | 7×24h 抓取 Product Hunt、Hacker News 等信号源，AI 自动评分排序 |
| **AI 评分系统** | 基于 DeepSeek API，从热度、商业价值、创新度、本地化潜力 4 维度量化评分 |
| **复刻工坊** | 一键调用 TRAE IDE 生成可运行 MVP，支持本地化改造和自定义配置 |
| **商业画布** | 基于信号自动生成完整商业模型画布、SWOT 分析、30 天行动清单 |
| **公开日志** | Build in Public 实时流，记录从信号发现到商业变现的全过程 |

## 2. Demo 创作思路

### 灵感来源

在独立开发者的日常工作中，最大痛点不是"不会写代码"，而是**"不知道做什么"**。每天有大量优秀产品在 Product Hunt、Hacker News 等平台涌现，但信息过载让开发者难以高效筛选有价值的方向。

### 想解决的问题

1. **信息过载**：每天数百个新产品上线，无法高效筛选
2. **评估困难**：缺乏量化标准判断一个创意的商业可行性
3. **落地门槛高**：从创意到 MVP 有大量重复性工作
4. **本地化盲区**：海外优秀产品如何适配国内市场缺乏指导

### 为什么做这个方向

在 AI 工具爆发式增长的背景下，"创意发现 → 商业评估 → 快速落地" 的完整链路存在巨大机会。SparkForge 不只是做一个信号聚合器，而是打通从灵感到产品的全流程，让独立开发者用 30 秒就能完成从发现创意到生成 MVP 的闭环。

## 5. 技术架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js 15 + React 19 | App Router 架构，服务端渲染 |

## 6. 开发心得与踩坑记录

### 踩坑1：Supabase 客户端在浏览器端报错
- **问题**：初始使用服务端 Supabase 客户端在客户端组件中初始化，导致 `supabaseKey is required` 错误
- **解决**：改为懒加载模式，区分服务端和客户端实例，客户端仅在浏览器环境初始化

### 踩坑2：DeepSeek API 余额不足
- **问题**：开发过程中 DeepSeek API 配额耗尽，导致评分系统不可用
- **解决**：实现了启发式评分作为后备方案，确保即使 API 不可用也能给出合理评分

### 踩坑3：Vercel Cron Job 频率限制
- **问题**：Hobby 账户只支持每天执行一次的 Cron Job，配置为每小时会部署失败
- **解决**：将抓取和评分任务改为每天凌晨执行，确保免费账户可用

### 踩坑4：API 认证不匹配
- **问题**：服务端订阅 API 添加了 Bearer token 认证，但客户端未携带 token
- **解决**：在客户端获取 Supabase session token，请求时携带 Authorization 头
