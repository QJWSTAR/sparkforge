# SparkForge 初赛提交差距分析 - Product Requirement Document

## Overview
- **Summary**: 基于 TRAE AI 创造力大赛初赛参赛指南，对 SparkForge 项目当前状态与初赛提交要求进行差距分析，制定详细工作步骤清单
- **Purpose**: 确保项目在 7月15日 截止前满足初赛提交的所有必要条件，最大化入围复赛概率
- **Target Users**: 参赛者（项目作者）、TRAE 大赛评审团

## 背景

### 初赛关键信息
- **阶段**: 初赛（2026年6月16日 – 7月15日）
- **前提**: 报名审核已通过
- **流程**: 报名通过 → 用 TRAE 做出可体验 Demo → 按模板发布初赛 Demo 帖 → 评审 → 7月21-23日公示
- **评审4维度**（各25%）: 创意与价值、完成度与体验、TRAE 实践深度、文档与表达
- **晋级名额**: 专业评审通道 300 席 + 抖音人气通道 50 席

### 初赛 Demo 帖必须包含
1. 标签（赛道四选一：生活娱乐/学习工作/社会服务/硬件交互）
2. Demo 简介（产品形态、核心用户、主要功能 + 截图）
3. Demo 创作思路（灵感来源、痛点、方向判断）
4. Demo 体验地址（公开可访问链接 或 HTML 文件 或 演示视频）
5. TRAE 实践过程（≥3张开发截图 + ≥3个 Session ID）
6. 报名帖链接

## Goals
- 完成 Vercel 部署，获得公开可访问体验链接
- 确认报名审核状态
- 撰写并发布初赛 Demo 帖
- 收集 Session ID 和开发截图
- 确保核心功能（信号抓取、AI评分、数据展示）可正常体验
- 最大化 4 个评审维度的得分潜力

## Non-Goals (Out of Scope)
- 新增功能开发（如实际 TRAE IDE 调用、Twitter 集成等）
- 复赛阶段的功能完善
- 抖音人气通道运营（作为可选项，非必须）

## Functional Requirements
- **FR-1**: Vercel 部署成功，体验链接可公开访问
- **FR-2**: 核心功能可正常体验（信号浏览、评分查看、详情页）
- **FR-3**: 初赛 Demo 帖撰写完成，包含所有必需部分
- **FR-4**: Session ID 收集（≥3个）
- **FR-5**: 开发截图准备（≥3张）
- **FR-6**: 赛道确认和标签选择

## 评审维度对齐分析

### 维度1: 创意与价值（25%）
- **优势**: SparkForge 解决"创意发现→产品落地"的真实痛点，差异化强
- **风险**: 创意方向需在 Demo 帖中清晰表达
- **建议**: 在帖中强调"从信号到 MVP 的一站式"差异化价值

### 维度2: 完成度与体验（25%）
- **优势**: 9个页面、7个API、完整前后端，功能闭环完整
- **风险**: 复刻工坊和商业画布为模拟数据，可能影响体验
- **建议**: 确保核心路径（首页→雷达→信号详情）流畅可体验；模拟功能标注为"Demo演示"

### 维度3: TRAE 实践深度（25%）
- **当前状态**: 项目由 TRAE 开发完成，有 Session ID 可证明
- **风险**: Session ID 未专门收集和整理
- **建议**: 整理出最关键的 3-5 个 Session ID，展示架构设计、核心功能开发、问题排查等关键节点

### 维度4: 文档与表达（25%）
- **当前状态**: 未撰写
- **风险**: 这是最大的差距项
- **建议**: 严格按模板撰写，配图丰富，过程详细

## Constraints
- **截止日期**: 2026年7月15日
- **Vercel**: 免费账户（Hobby），Cron Job 每天一次
- **环境变量**: 7个核心变量需在 Vercel 配置

## Assumptions
- 报名审核已通过（需确认）
- TRAE 对话记录中保留了足够的 Session ID
- Vercel 部署不会遇到额外技术问题

## Acceptance Criteria

### AC-1: Vercel 部署成功
- **Given**: 代码已推送到 GitHub
- **When**: 在 Vercel 配置环境变量并部署
- **Then**: 获得公开可访问的体验链接，核心功能正常
- **Verification**: `programmatic`

### AC-2: 初赛 Demo 帖发布
- **Given**: 所有材料准备就绪
- **When**: 在 TRAE 社区初赛专区发帖
- **Then**: 帖子包含所有必需部分（简介、思路、体验链接、实践过程、截图、Session ID）
- **Verification**: `human-judgment`

### AC-3: Session ID 收集
- **Given**: TRAE 对话记录
- **When**: 整理关键对话的 Session ID
- **Then**: 至少 3 个 Session ID，覆盖架构设计、核心开发、问题排查等关键节点
- **Verification**: `human-judgment`

### AC-4: 开发截图准备
- **Given**: 开发过程
- **When**: 截取关键步骤截图
- **Then**: 至少 3 张截图，展示 TRAE 开发过程
- **Verification**: `human-judgment`

### AC-5: 体验链接可用
- **Given**: Vercel 部署完成
- **When**: 访问体验链接
- **Then**: 能正常浏览信号、查看评分、查看详情，核心功能可体验
- **Verification**: `programmatic`
